02/ Admin overhead
     How can a self-service platform reduce support requests, staff workload, and costs?
Group sticky noting session on Miro


UVU’s four legacy systems had created a maze of duplicate logins and desktop dependencies. During a collaborative 3-day sprint with IT directors and stakeholders, we used Miro to map every critical user flow—from registration to financial aid—identifying which features absolutely had to make it into the MVP. The whiteboard sessions revealed surprising priorities: while administrators assumed students wanted flashy new tools, the data showed they just needed reliable access to existing services on mobile.

Security Barriers Students avoided old systems because they were clunky and confusing. Our mobile-first approach replaced four disjointed logins with a secure, intuitive dashboard—proving that good UX and strong security aren’t mutually exclusive.
Prototyping & User Testing
Refining Through Real Student Feedback

We evolved UVU’s wireframes into high-fidelity Figma prototypes, perfecting core flows like course registration and task management. This created a functional foundation for real-world testing.Over four days of remote sessions with 30+ students, we combined quantitative metrics (75% task completion) with qualitative feedback. The data revealed key opportunities—like simplifying ambiguous navigation labels and boosting visibility of deadline alerts—that shaped our final design.

Visual design
A Lightweight, Student-First Design

We crafted an interface that balanced UVU’s brand identity with raw performance. Instead of heavy assets or time-intensive gradients, we focused on what actually mattered—speed and usability. The palette leaned on UVU’s signature deep green for key actions, set against clean backgrounds with ample whitespace. We designed for the smallest modern mobile size (320px) first and used those constraints to make it responsive on larger phones.

Accessibility wasn’t an afterthought. When testing revealed strong demand for dark mode, we implemented it with WCAG AA contrast ratios, reducing eye strain without compromising the app’s sub-1.5s load time. Every design decision answered two questions: Does this help students complete tasks faster? and Could we ship it tomorrow? The result was an MVP that looked unmistakably UVU without a single unnecessary byte.

From Confusion to Clarity

Testing revealed students struggled with the original flow’s inconsistent navigation and ambiguous labels (“Register” vs. “Schedule”). Applying Hick’s Law, we:

Reduced options by 60%, eliminating backtracking

Renamed actions to match student mental models

Aligned flows with academic deadlines

A follow-up test with students confirmed the refined flow felt more intuitive—click paths shortened by 40%, and frustration signals disappeared.
